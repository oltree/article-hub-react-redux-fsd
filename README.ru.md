## Обзор проекта

Article Hub - полнофункциональная контентная платформа, где пользователи могут:

- Просматривать и читать статьи с блоками текста, кода и изображений
- Писать и редактировать собственные статьи
- Оценивать статьи по звёздочной системе
- Оставлять комментарии и читать комментарии других
- Просматривать рекомендованные статьи
- Управлять своим профилем
- Переключаться между светлой и тёмной темой
- Использовать интерфейс на русском или английском языке

Приложение включает: панель администратора для пользователей с повышенными правами (MANAGER / ADMIN), защищённую маршрутизацию с ролевым контролем доступа, а также двойную дизайн-систему, управляемую runtime feature flag для поэтапного перехода на новый интерфейс.

## Архитектура

Проект строго следует методологии **Feature Sliced Design (FSD)** - масштабируемой модульной архитектуре, которая организует код по бизнес-доменам и обеспечивает чёткие границы между слоями.

```
src/
├── app/          # Инициализация приложения (провайдеры, глобальные стили, роутинг)
├── pages/        # Компоненты страниц (уровень маршрутов)
├── widgets/      # Крупные самодостаточные UI-блоки (Navbar, Sidebar, Фильтры…)
├── features/     # Пользовательская функциональность (логин, рейтинг, комментарии…)
├── entities/     # Бизнес-сущности (User, Article, Comment…)
└── shared/       # Переиспользуемые утилиты, UI-кит, хуки, API-клиент
```

### Направление импортов (только вниз)

```
app → pages → widgets → features → entities → shared
```

Ни один слой не может импортировать из слоя выше. Это гарантирует изолированность, независимую тестируемость и возможность замены любого модуля без побочных эффектов.

### Ключевые архитектурные решения

**Динамическая загрузка редьюсеров** - Redux-слайсы монтируются по требованию через `DynamicModuleLoader`. В памяти находятся только редьюсеры, нужные текущей странице, что уменьшает размер начального бандла.

**Feature flags** - Утилита `toggleFeatures({ name, on, off })` позволяет поэтапно выкатывать новую функциональность без разветвления маршрутов или дублирования кода. Флаг `isAppRedesigned` переключает всю дизайн-систему в runtime.

**Двойная дизайн-система** - В `shared/ui/` сосуществуют `deprecated/` (оригинальный дизайн) и `redesigned/` (новый). Feature flag решает, какой именно рендерится - пользователь никогда не видит «полумигрированный» интерфейс.

**Code splitting** - Каждая страница лениво загружается через `React.lazy()` по соглашению об именовании `.async.tsx`. Редьюсеры следуют той же схеме загрузки по требованию.

**Public API exports** - Каждый FSD-модуль раскрывает только то, чем намерен поделиться, через свой `index.ts`. Внутренние детали реализации недоступны для импорта извне.

## Функциональность

### Аутентификация и авторизация

- Форма входа по логину и паролю с состояниями загрузки и ошибки
- Токен хранится в `localStorage`, автоматически добавляется в каждый запрос через Axios-интерсептор и `prepareHeaders` RTK Query
- Компонент `RequireAuth` защищает все приватные маршруты на уровне роутера
- Ролевой доступ: обычные пользователи, MANAGER, ADMIN
- Сессия автоматически восстанавливается из `localStorage` при каждом запуске приложения (thunk `initAuthData`)

### Система статей

- Пагинированный список статей с бесконечной прокруткой (Intersection Observer)
- Фильтрация по типу статьи: IT, Экономика, Наука, Все
- Сортировка по дате, просмотрам или рейтингу; переключение по возрастанию / убыванию
- Два режима отображения: сетка карточек и компактный список
- Детальная страница статьи с тремя типами блоков: **Текст**, **Код** (с подсветкой синтаксиса), **Изображение**
- Создание и редактирование статей через блочный редактор (только для авторизованных)
- Блок рекомендаций с похожими статьями

### Социальность и вовлечённость

- Оценка статей по шкале 1–5 звёзд с полем для отзыва
- Раздел комментариев с инлайн-формой добавления
- Страница профиля пользователя: аватар, имя, город, страна, валюта, возраст
- Редактируемая карточка профиля с валидацией полей
- Колокольчик уведомлений со счётчиком непрочитанных

### Панель администратора

- Доступна только пользователям с ролью MANAGER или ADMIN - проверяется и на уровне роутера, и на бэкенде
- Предоставляет UI управления контентом системного уровня

### UX и доступность

- Переключатель светлой / тёмной темы, выбор сохраняется в `localStorage`
- Переключатель языка (Русский ↔ English) с автоматическим определением языка браузера при первом визите
- Кнопка «Прокрутить наверх», появляющаяся после скролла вниз
- Позиция прокрутки сохраняется при возврате на страницу со списком статей
- Прилипающий сайдбар и тулбар с фильтрами на странице списка статей
- Адаптивные макеты на базе компонентов `StickyContentLayout` и `MainLayout`

## Быстрый старт

### Требования

- Node.js ≥ 16
- npm ≥ 7

### Установка зависимостей

```bash
npm install
```

### Запуск mock API-сервера

Бэкенд эмулируется через json-server на порту **8000** с искусственной задержкой 800 мс для имитации реальных сетевых условий.

```bash
npm run start:dev:server
```

### Запуск dev-сервера

**Webpack** (основной, порт 3000):
```bash
npm run start:dev
```

**Vite** (быстрее при холодном старте):
```bash
npm run start:dev:vite
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

### Тестовые учётные данные

Можно использовать любого пользователя из `/json-server/db.json`. Пример:
```
username: testuser
password: 123
```

## Тестирование

В проекте **три взаимодополняющих уровня тестирования**:

### Unit-тесты - Jest + React Testing Library

Тесты хранятся рядом с исходными файлами (`Component.test.tsx`). SCSS-модули и SVG замоканы. Алиасы модулей (`@/`) разрешаются автоматически. Матчеры из `@testing-library/jest-dom` доступны глобально. После каждого запуска генерируется HTML-отчёт в папке `reports/`.

```bash
npm run test:unit
```

### E2E-тесты - Cypress

Полные пользовательские сценарии в `/cypress/e2e/` покрывают: аутентификацию, просмотр списка статей, создание статьи, редактирование профиля и другое. Переиспользуемые команды и фикстуры находятся в `/cypress/support/`.

```bash
npm run test:e2e
```

### Визуальная регрессия - Loki

Loki рендерит Storybook-стори в headless Chrome в Docker-контейнере и сравнивает пиксель-в-пиксель результат с утверждёнными эталонными скриншотами. Тестируются два viewport: **desktop** (1366×768) и **мобильный** (iPhone 7).

```bash
npm run test:ui        # сравнить с эталонами
npm run test:ui:ok     # принять новые скриншоты как эталоны
npm run test:ui:ci     # строгий режим (ошибка при любом отличии)
```

## CI/CD

Пайплайн GitHub Actions (`.github/workflows/main.yml`) запускается на каждый push и pull request в ветку `master`:

1. `npm ci` - чистая установка зависимостей
2. `npm run build:prod` - production-сборка
3. `npm run lint:ts` - линтинг TypeScript
4. `npm run lint:scss` - линтинг SCSS
5. `npm run test:unit` - unit-тесты

Все шаги используют `if: always()`, поэтому ошибка на одном шаге не скрывает ошибки последующих. Пайплайн работает на **Node.js 21.x**.

Pre-commit хуки (Husky + lint-staged) обеспечивают соблюдение тех же правил линтинга и форматирования локально перед каждым коммитом.

## Что демонстрирует этот проект

Это не учебный проект. Кодовая база отражает реальные практики enterprise frontend-разработки:

| Область | Реализация |
|---|---|
| **Масштабируемая архитектура** | Feature Sliced Design с принудительной изоляцией слоёв через ESLint |
| **TypeScript** | Типизированные thunk-и, селекторы, ответы API, пропсы компонентов |
| **Управление состоянием** | Слайсы Redux Toolkit, RTK Query endpoints, динамическая инъекция редьюсеров |
| **Культура тестирования** | Три уровня тестов: unit, E2E и визуальная регрессия |
| **Качество кода** | ESLint (Airbnb + FSD-правила), Stylelint, Prettier, pre-commit хуки Husky |
| **Производительность** | Code splitting, ленивые маршруты, ленивые редьюсеры, debounce, бесконечная прокрутка |
| **Интернационализация** | Полная двуязычная поддержка с файлами переводов по пространствам имён |
| **Миграция дизайн-системы** | Двойной UI-кит с runtime feature flag - никаких переписываний «с нуля» |
| **Конфигурация сборки** | Ручная конфигурация Webpack 5 + Vite как zero-config альтернатива |
| **CI/CD** | Автоматизированный пайплайн сборки, линтинга и тестирования на каждый коммит |
| **Документация компонентов** | Storybook-стори с моками API и вариантами по теме и языку |
